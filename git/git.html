Lab: Intro To Git

Introduction

Today we will explore the leading version control software, Git, and we get some more practice with Ansible and bash!

Theory

Here's and excellent video from MIT covering Git. The first 30 minutes should be more than enough to get started.

Developers use Git to track changes to software and to be able to 'roll back the clock' to view old versions of files. Increasingly admins are using Git to track configuration files, particularly Ansible playbooks.

Even admins who choose not to use Git to version their own files, will often have to use Git to install and update software. For this lab we will be exploring this strategy. The development team responsible for the TRS application from the 3-Tier lab has published the application on GitHub, a public Git server. The project can be found at https://github.com/profjamesmohawk/trs

Overview

Today will will focus on using Ansible with Git to install and configure the TRS application. Frankly the developers have done a terrible job of creating installation scripts and documentation. At work we might refuse to proceed until the developers provide installation documentation.

Requirements

  • Create an Ansible playbook, trs.yml, to install the TRS application and DB on a single new server.
  • Create an Ansible playbook, trs2.yml to install the TRS application on two servers, one for the database and one for the web server.
