Lab: Test Systems Getting Started

Introduction

We have five servers at our disposal for COMP-10097. They are rack mounted in the Mohawk data centre. We can get access via their remote console functionality.

Since there are more students than servers, you must reserve a system before you use it. Reservations will be handled by the Trivial Reservation System. This URL is only accessible from the Mohawk network, as are the systems. The systems are denoted as sut_[1-5], short for "system under test".

Your user name is "<first_name><last_initial'> (Fred White's user name is fredw).
Your passwd is your student number.

For this lab, your task is to reserve a system, connect to the remote console, and install the operating system of your choice on the system.

Grading

To earn 5 work product marks you must demonstrate your running system to your professor.  Write up and submit a short procedure describing what you had to do to get everything working.

Tips

This is an exercise in discovery and trouble shooting, but we will give you some tips.

  • The TLS/SSL certificates on the management cards do not have valid signatures. Use a browser that lets you ignore this.
  • You will need Java to run the remote console.
  • The java file supplied by the remote management card is not named correctly, for it to be 'auto-run'. Find a browser plugin that will modify the name during the download.
