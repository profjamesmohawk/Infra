Lab: Private Docker Registry

Introduction

To date we have used the Docker public registry. This is fine for testing, but not appropriate for production. Today we will deploy our own private registry.

Part A: Provision the lab systems

Design your own 'test' lab with the systems you need.

Part B: Run an 'open' local registry

Our first task is to install and run a local, completely open registry. Fortunately, it is packaged as a Docker container, so this should be quick and easy.

  • Follow this tutorial up to "Storage Customization". It is important that our registry store images somewhere other than 'inside' the container, we don't want to loose any.

Part C: Run an externally-accessible registry

For production, we need to have a registry that is accessible to all the nodes on our network.

  • Most of what you need to know is contained in the tutorial from Part B
  • Use as self-signed certificate (Docker calls this "insecure")
  • Create two users: developer and admin
  • Test your access.

Part D: Grading

  • Demonstrate your registry to your instructor.
  • Submit a short document to eLearn answering the question, "What are the security risks of using a self-signed certificate?"
