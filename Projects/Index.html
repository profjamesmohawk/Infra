COMP-10097 Project Inventory

Introduction

This document contains a list of suggested projects. These are just suggestions, any relevant project is acceptable. Be sure to check with your instructor before starting any project for which you want credit. The number of points awarded for any project will depend on scope and quality of your solution. A given project could vary greatly in scope from quick proof of concept (POC) to production ready. Discuss scope with your instructor before starting the project to avoid surprises.

A project's scope may also grow after it has started or even completed. A proof of concept could lead to a product quality implementation. A working system might lead to a performance testing project. Don't hesitate to talk to your instructor about scope.

This is a living document, check back regularly for updates.

A Replacement for "Deepfreeze"

You should all be familiar with the Deepfreeze product used to protect the integrity of Mohawk's lab systems. It should be possible to build a similar system using visualization tools.

Consider a system in which a "locked-down" base O/S image boots and presents the user with a menu of immutable VM images to boot.

Likely Scope: 5-10 points

VDI Environment

Virtual Desktop Infrastructure (VDI) allows "think" clients to run desktop O/S images hosted on servers. This is usually sold as a way to protect corporate data, as all data remains safe in the data centre. Citrix is the big player in this space. Your challenge is to build a VDI environment. The easy, low mark way would be to install a demo version of a commercial tool like Citrix and perform some functionality and performance tests. A far more interesting task is to build your own using more basic tools.

Some things to consider:
  • What image O/S do we need to support? (Ubuntu is likely a better choice then Window$ because of licensing)
  • What kind of clients to we need to support? (A Windows or Linux client might be good choices. Can you present the desktop in a browser? On a phone?)
  • Should each desktop be a VM or a container?
  • Does each user get a unique desktop on which they can install software, or does everyone get the same image?
  • How many concurrent users can one node support?
  • How much network bandwidth is required per user?
  • How much server memory and disk space is required per user?
Likely Scope: 10 - 50 points

Web Hosting Infrastructure - Many Small Sites

M&S consulting wants to get into the hosting business. Bullwinkle has boldly stated, "If Amazon can do it, we can do it better." Rather than challenging AWS head on, consider building an environment to host a large number of small web sites.

Some things to consider:
  • What server applications do your customers need (php, mysql, ...)?
  • How will you bill?
  • Do you need a load balancer?
  • Should your server farm scale to more than one server?
  • How many sites can one server support?
  • How much traffic can one server support?
Likely Scope: 10 - 30 points

Web Hosting Infrastructure - One Large Site

Many web sites need more power than one server can deliver. Design and implement infrastructure to allow many nodes to satisfy requests to a singe web site.

Some things to consider:
  • What sort of load balancer will you use?
  • How do you add or remove a node from the farm?
  • How do you tell how busy your farm is?
Likely Scope: 10 - 30 points

Unix Fundamentals for Remote Users

The current offering of Unix Fundamentals requires students to run a CentOS VM containing course files. This works very well for on-site students, but does not support remote, on-line only students. Design a server based solution to present terminal window access to a remote students.

Some things to consider:
  • Can you automate creation of student images/VMs/containers? (won't be attractive if we must add students one at a time)
  • Should the image be immutable? (this would help with support)
  • How many students can be supported on one server?
  • What are the student's client software requirements. (ssh client, can you limit requirements to a browser?)
Likely Scope: 5 - 15 points

Home Brew Container Manager GUI

This one is for the aspiring coders. Docker's commercial container manager, Universal Control Panel, is "just" a nice wrapper on functionality available from the command line. Write a web application to perform image and container management.

Some things to consider:
  • What operations do you need to support?
  • How do you want to limit access to the functionality (different levels of access: admin, operator)?
  • What web development tools do you want to use?
Likely Scope: 10 - 50 points

Network Billing Tool

You want to bill your customers, or roommates, based on the network traffic they generate. Build a system to do so.

Some things to consider:
  • Is your tool slowing the network?
  • How friendly are the reports generate?
Likely Scope: 5-10 points

Low Level Performance Test

Bullwinkle and Rocky have on ongoing argument about how much of a performance penalty is involved with various virtualisation tools. The best way to know is to test. Devise and execute some tests to quantify the overhead for three main categories: CPU, Disk I/O, network I/O for one or more VM environments. This a good one to do at home or in the class room with VirtualBox.

Some things to consider:
  • What tools will you use?
  • How will you ensure that your results are repeatable?
  • How will you present your results?
Likely Scope: 5-10 points

High Level Performance Test

Once a system is build we need to conduct "end to end" performance tests. For VDI projects this might be as 'simple' as organizing a group of your peers to use the application simultaneously. Web applications can be tested with tools ranging from wget to commercial load generators. This project is an opportunity to be a consultant to your peers. If you develop a good testing methodology, you can help other teams test their projects and earn "Good Employee" points and get credit for your time spent.

Likely Scope: 5-10 points
